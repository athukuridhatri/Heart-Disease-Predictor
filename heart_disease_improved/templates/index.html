💡 INTERVIEW TALKING POINTS

  • Dataset: UCI Cleveland Heart Disease — 278 clean samples, 13 clinical features, binary target (disease / no disease)
  • Algorithm: Logistic Regression — outputs probability via sigmoid; coefficients directly show feature impact direction
  • Preprocessing: StandardScaler normalises all features to mean=0, std=1 — critical to prevent large-value bias in LR
  • Split: Stratified 80/20 split — preserves class ratio in both train and test sets
  • Metrics: Accuracy, Precision, Recall, F1, ROC-AUC — Recall especially important (minimising false negatives)
  • Why LR? Simple, interpretable, fast to train; each coefficient shows which features push risk up or down
  • Stack: Python · Flask · scikit-learn · pandas · numpy · matplotlib · HTML/CSS/JS
